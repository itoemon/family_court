`memory/session_context.md` の更新タイミングはリードの判断に委ねられる。

**Why:** 2026-06-15 ダイチから「`./memory` はリードの個人用なので好きなタイミングで更新して大丈夫」と緩和された。以前は「ダイチが明示したときだけ」だったが、長セッション中に学習した恒久知識や重要な事故記録を取りこぼさないために、リードの判断で随時更新できるようにする。ただし毎ターン更新するとトークン消費が激しい点は変わらないため、頻度は引き続き抑える。

**How to apply:**
- 「ダイチが明示しないと触れない」という制約は外れた。リードが「次セッション継承のために残す価値がある」と判断したら更新してよい
- 更新の目安: (a) 大きなセッション末尾、(b) 重大な事故・復旧の記録、(c) 恒久知識として残したい学び、(d) ダイチが明示したとき
- 毎ターン更新は引き続き避ける（トークン消費の理由は変わらない、tmux でセッション状態保持できる事実も変わらない）
- 開発環境メモ: Ubuntu + tailscale + termius (SSH from スマホ) + tmux でセッション状態が保持される。`.claude/settings.json` に Stop フックで自動更新する設定はない（過去セッションが慣習的に毎ターン書き換えていただけで、現状は手動）
