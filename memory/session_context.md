# セッション引き継ぎ

新しいチャットを開いたら、リードはこのファイルを読んで前回の状況を把握する。
セッション終了時またはひと区切りついたタイミングで更新する。

---

## 最終更新: 2026-05-19

### このセッションでやったこと

**開発体制の整備（メイン）**

- エージェント呼び名を決定: リード / アーキ / ビルド / オーディ / コパ
- `docs/` ナレッジDB を構築（agents.md, pipeline.md, decisions/ 5件, knowledge/, reference/）
- `scripts/agents.sh` / `scripts/pipeline.sh` を実装してパイプラインを動かせるようにした
- GitHub に push・Vercel デプロイ済み・gh CLI もインストール済み
- GitHub Copilot（コパ）を Ruleset で PR 自動レビューに設定
- `.github/copilot-instructions.md` にコパへのレビュー観点を定義

**実際に動かした機能**

- 共通ヘッダー・フッターをパイプライン経由で実装・マージ（PR #1）
- バックログの MEDIUM 指摘を修正・コパレビューを経てマージ（PR #2）

**仕組みとして整備したもの**

- `docs/backlog.md`: オーディの未修正指摘を蓄積。リードが次セッション冒頭でダイチに共有する
- `docs/pipeline.md` に「誰が指摘を直すか」の判断基準を追記（誤記→リード、設計変更→アーキ）
- 各エージェントのプロンプトにキャラクター定義を追加

### 覚えておくべき判断・経緯

- ビルドの作業場所は `src/` にまとめず、ルートに `app/`, `lib/` のまま（Next.js のデフォルト構造を維持）
- エージェントのディレクトリ権限はプロンプトベースの制御（ファイルシステムレベルの強制なし）
- コパの指摘は `gh api` で取得できる（GitHub を開かなくてよい）
