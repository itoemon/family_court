# ADR-002: 認証戦略

## 決定内容

- **提案者（原告）**: アカウント必須
- **反対者（被告）**: アカウントログイン または ゲスト（名前のみ）を選択可能

## 理由

このアプリは恋人・夫婦・家族が使うことを想定している。
話し合いを始めたい人（提案者）はAPIキーを登録するためにアカウントが必要だが、
招待される側（反対者）にアカウント登録を強制するとハードルが高くなり、利用率が下がる。

Google Docs と同じ考え方で「作った人がコストを負担し、招待された人はリンクだけで参加できる」設計にした。

## 代替案

- 両方アカウント必須: 被告の離脱率が上がるため却下
- 両方ゲスト可: APIキー管理が複雑になり、悪用リスクもあるため却下

## 結果

被告が既存アカウントを持っていれば履歴が残せる。持っていなければゲストとして気軽に参加できる。
将来的にゲスト参加後にアカウント作成した場合の履歴紐付けを検討余地あり。
