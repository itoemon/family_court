# ADR-004: AI 接続方式（BYOK）

## 決定内容

ユーザーが自分の AI API キーを持ち込む BYOK（Bring Your Own Key）方式を採用する。
アプリ側で共有 API キーは持たない。

## 理由

- API コストをユーザーが負担することで、アプリの運営コストをゼロにできる
- 共有キーを持つ場合、悪用・乱用のリスクが高く、レート制限への対処も複雑になる
- 夫婦・恋人間のプライベートな会話を扱うため、キーの所有者（原告）が全コストを把握・管理できる設計が適切

## セキュリティ

- API キーはブラウザに平文で送らず、サーバーサイドのみで復号・使用する
- DB には AES-256-GCM 暗号化済みの値のみ保存（ADR-003 参照）
- 判決生成（`/api/cases/[id]/verdict`）のみキーを復号する

## 代替案

- **アプリ共有キー**: 運営コスト・悪用リスクがあるため却下
- **ユーザーが毎回入力**: UX が悪く、平文がブラウザメモリに残るリスクがあるため却下

## 結果

現在は Claude（Anthropic）のみ対応。将来的に OpenAI / Gemini などへの拡張も同じ BYOK 方式で追加可能。
