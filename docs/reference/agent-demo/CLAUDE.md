# プロジェクトルール

## 【最重要】操作範囲の制限
- このプロジェクトのルートディレクトリ配下のみ操作可能
- 上位ディレクトリ（../）へのアクセスは一切禁止
- システムディレクトリへのアクセスは一切禁止

## ディレクトリ構成
- `knowledge/requirements/` - 要件定義書
- `knowledge/design/` - 設計書
- `knowledge/audit-log/` - レビュー結果
- `src/` - ソースコード

## エージェントルール
- 設計者は必ず `knowledge/requirements/` を読んでから設計書を書くこと
- 実装者は必ず `knowledge/design/` を読んでからコードを書くこと
- 審査者は必ず `src/` のコードを読んでから `knowledge/audit-log/` にレポートを書くこと
- 各エージェントは自分の役割以外の作業をしないこと
