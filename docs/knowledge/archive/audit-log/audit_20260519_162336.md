## 指摘一覧

### [MEDIUM-001] logout() で signOut() 失敗時のセッション残存リスク（`app/actions/auth.ts`:8）

- **内容**: `supabase.auth.signOut()` は `{ error }` を返す可能性があるが、戻り値を検査せず直後に `redirect('/')` を実行している。`signOut()` がエラーを返しても redirect は成立するため、ユーザーはログアウトしたと認識するが、Supabase 側のサーバーセッションが残存する可能性がある。
- **修正案**: `signOut()` の戻り値を検査し、エラー時はリダイレクト前にログを記録するか例外をスローしてセッション残存を防ぐ。

  ```ts
  export async function logout() {
    const supabase = await createSessionClient()
    const { error } = await supabase.auth.signOut()
    if (error) {
      // 必要に応じてサーバーログへ記録
      console.error('signOut failed:', error.message)
    }
    redirect('/')
  }
  ```

### [LOW-001] Header.tsx で getUser() のエラーオブジェクトを無視（`app/components/Header.tsx`:6）

- **内容**: `const { data: { user } } = await supabase.auth.getUser()` は `error` フィールドを捨てている。Supabase はネットワーク障害等でも `data.user = null` を返す設計のため即時クラッシュはしないが、エラー発生時に原因が追えず障害調査が困難になる。
- **修正案**: `error` を分離して受け取り、ログ記録またはモニタリングに渡す。

  ```ts
  const { data: { user }, error } = await supabase.auth.getUser()
  if (error) console.error('getUser error:', error.message)
  ```

---

### [LOW-002] 設計書の `createSessionClient()` に `await` 欠落（`docs/knowledge/design/design.md`）

- **内容**: 設計書「Server Action（`app/actions/auth.ts`）」節のコードサンプルでは `const supabase = createSessionClient()` と `await` なしで記載されているが、`createSessionClient` は `async function`（`await cookies()` を内部で使用）であるため、`await` なしでは Promise が代入されてしまい `supabase.auth.signOut()` が実行時エラーになる。実装コードは正しく `await` を付けているため動作上の問題はないが、設計書の誤記が後続の変更で混乱を招くリスクがある。
- **修正案**: 設計書のサンプルコードを `const supabase = await createSessionClient()` に修正する。

### [LOW-003] Footer.tsx の著作権年ハードコード（`app/components/Footer.tsx`:4）

- **内容**: `© 2026 家庭裁判所` と年号をリテラルで埋め込んでいる。毎年手動更新が必要となる保守上の負債。
- **修正案**: `new Date().getFullYear()` を利用して動的生成する（Server Component のため実行時評価が可能）。

  ```tsx
  <footer className="bg-stone-100 text-stone-400 text-sm text-center py-4">
    © {new Date().getFullYear()} 家庭裁判所
  </footer>
  ```
