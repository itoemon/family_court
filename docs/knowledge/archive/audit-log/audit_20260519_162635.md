## 指摘一覧

### [MEDIUM-001] logout() で signOut() のエラーが握り潰される（app/actions/auth.ts:8）
- **内容**: `supabase.auth.signOut()` は `{ error }` オブジェクトを返すが、現在の実装ではその戻り値を検査していない。Supabase クライアントはネットワークエラー等をスローせず `error` プロパティに格納して返す設計であるため、サインアウトが失敗してもユーザーは `/` へリダイレクトされる。結果として、ユーザーはログアウトしたと認識するが、サーバー側セッションが残存する可能性がある。
- **修正案**:
  ```ts
  export async function logout() {
    const supabase = await createSessionClient()
    const { error } = await supabase.auth.signOut()
    if (error) {
      // エラーをログに記録し、ユーザーに通知するか再試行する
      throw new Error('ログアウトに失敗しました')
    }
    redirect('/')
  }
  ```

### [MEDIUM-002] Header の非同期処理に Suspense 境界がない（app/layout.tsx:32）
- **内容**: `Header` は `async` Server Component であり、`createSessionClient()` → `auth.getUser()` という非同期 I/O を含む。`layout.tsx` ではこの `<Header />` が Suspense でラップされていないため、Supabase への通信が遅延した場合にページ全体のレンダリングがブロックされる。攻撃者が Supabase エンドポイントへのレート制限を意図的に引き起こす状況では、全ページの応答遅延（DoS 的影響）につながりうる。
- **修正案**: `app/layout.tsx` で `<Header />` を `<Suspense>` でラップし、フォールバックを提供する。
  ```tsx
  import { Suspense } from 'react'

  <body className="min-h-screen flex flex-col bg-white text-stone-900">
    <Suspense fallback={<div className="h-12 bg-stone-50 border-b border-stone-200" />}>
      <Header />
    </Suspense>
    <main className="flex-1">{children}</main>
    <Footer />
  </body>
  ```

### [LOW-001] 設計書の logout() サンプルコードに `await` 漏れ（設計書: Server Action セクション）
- **内容**: 設計書の Server Action サンプルコードが `const supabase = createSessionClient()` と記載されており `await` が欠落している。実装は正しく `await createSessionClient()` としているため動作上の問題はないが、設計書と実装が乖離しており、将来の実装者が設計書を参照した際にバグを持ち込むリスクがある。
- **修正案**: 設計書の該当箇所を以下のように修正する。
  ```ts
  const supabase = await createSessionClient()
  ```

### [LOW-002] layout.tsx の `<main>` ラッパーが子ページと二重になりうる（app/layout.tsx:33）
- **内容**: `layout.tsx` が `{children}` を `<main className="flex-1">` でラップする設計になっている。既存または将来の子ページコンポーネントが独自に `<main>` タグを持つ場合、`<main>` がネストされた無効な HTML が生成される（HTML 仕様では `<main>` はページにひとつのみ許容）。スクリーンリーダー等の支援技術が誤動作する可能性があり、アクセシビリティ要件（恋人・夫婦・家族向けアプリとして幅広い利用者を想定）に反しうる。
- **修正案**: 子ページのコンポーネントが `<main>` を使用しないようコーディング規約を明文化する。または `layout.tsx` のラッパーを `<main>` から `<div>` に変更し、各ページが `<main>` を持つ構造に統一する。
