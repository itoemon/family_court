## 指摘一覧

### [LOW-001] open redirect ガードが backslash 系バイパスを考慮していない（`app/auth/login/page.tsx:37-39`）
- **内容**: 現状のガードは `rawNext.startsWith("/") && !rawNext.startsWith("//")` のみ。これは WHATWG URL 仕様で `\` を `/` と同等に扱う browser 挙動の影響を受け得るパターン（例: `?next=/\evil.com`、`?next=/%2fevil.com`、`?next=/.//evil.com`）を素通しさせる。
  - 本 PR では遷移手段が `router.push()` であり、Next.js router は受け取った文字列を内部パス前提で扱うため、現時点では外部ドメインへの全画面遷移は発生しない（事実上影響なし）。
  - しかし将来 `window.location.href = next` 等への変更や、Server-Side redirect 経路で `next` を流用する派生実装が入った瞬間、上記バイパスが open redirect として実利化する。task.md の監査観点 (3) は「念のため検証」を求めており、防御を残すなら強度を上げておくべき。
- **修正案**: 防御を URL パーサベースに置き換える（`router.push` 以外の経路でも一貫して安全）：
  ```ts
  const rawNext = searchParams.get("next");
  let next = "/";
  if (rawNext) {
    try {
      const u = new URL(rawNext, window.location.origin);
      if (u.origin === window.location.origin) next = u.pathname + u.search + u.hash;
    } catch { /* fallthrough → "/" */ }
  }
  router.push(next);
  ```
  もしくは現方式を残すなら最低限 `&& !rawNext.startsWith("/\\") && !rawNext.toLowerCase().startsWith("/%2f")` を追加する。

### [LOW-002] open redirect の負例テストが E2E spec に存在しない（`tests/e2e/auth-login.spec.ts:52-69`）
- **内容**: BUG-007-2 は内部相対パス `/history` のポジティブケースだけを検証している。task.md の監査観点 (3) で挙げられた外部 URL や protocol-relative URL を `next` に渡したときに `/` にフォールバックすることを保証する spec が無い。テスタ引き継ぎメモ「外部 URL の脆弱性は検出されず」は当該 case を実際に走らせて得た結論ではなく、コード見立てに過ぎないため、リグレッション検出力が弱い。
- **修正案**: 同 spec に以下 2 ケースを追加する（実装は LOW-001 修正後でも現行ガードでも fail しない範囲）。
  ```ts
  test('BUG-007-4: ?next=https://evil.example.com は無視され "/" に遷移', async ({ page }) => {
    await page.goto('/auth/login?next=https://evil.example.com');
    await page.fill('input[type="email"]', process.env.E2E_TEST_EMAIL_A!);
    await page.fill('input[type="password"]', process.env.E2E_TEST_PASSWORD_A!);
    await page.click('button[type="submit"]');
    await page.waitForURL('/', { timeout: 15_000 });
  });

  test('BUG-007-5: ?next=//evil.example.com は無視され "/" に遷移', async ({ page }) => {
    await page.goto('/auth/login?next=//evil.example.com');
    await page.fill('input[type="email"]', process.env.E2E_TEST_EMAIL_A!);
    await page.fill('input[type="password"]', process.env.E2E_TEST_PASSWORD_A!);
    await page.click('button[type="submit"]');
    await page.waitForURL('/', { timeout: 15_000 });
  });
  ```
