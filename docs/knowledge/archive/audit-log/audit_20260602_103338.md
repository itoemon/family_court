## 指摘一覧

### [LOW-001] `aria-controls` が DOM 上に存在しないメニュー ID を指している（`app/components/HeaderUserMenu.tsx:110`）
- **内容**: トリガボタンに `aria-controls={menuId}` を常時付与しているが、ドロップダウン本体（`id={menuId}` を持つ `<div role="menu">`）は `isOpen && (...)` でガードされているため `app/components/HeaderUserMenu.tsx:118-184` の通り閉じている間は DOM に存在しない。WAI-ARIA 1.2 の推奨では `aria-controls` の参照先は存在している必要があり、スクリーンリーダー実装によっては「制御先要素なし」と扱われたり警告を出すケースがある。エンドユーザーへの実害は限定的だが、SR で「メニューを開くボタン」のコンテキストが弱まる可能性がある。
- **修正案**: メニュー本体を常にレンダリングし `hidden` 属性 / `display:none` 相当のスタイル切り替えで開閉する、または `aria-controls` の付与を `isOpen` の間だけに限定する。前者の方が SR の整合性が高く、`mount/unmount` に伴うフォーカス挙動の差も解消できる（例: 常時 `<div id={menuId} role="menu" hidden={!isOpen}>...</div>`、トリガ側は現状の `aria-controls={menuId}` を維持）。

### [LOW-002] `role="menu"` 直下に `<form>` ラッパが入っており WAI-ARIA のメニュー構造に違反（`app/components/HeaderUserMenu.tsx:152-160`）
- **内容**: `app/components/HeaderUserMenu.tsx:119-124` の `<div role="menu">` の直接の子に `<form action={logout} onSubmit={close}>` が含まれており、その内側の `<button role="menuitem">` がログアウト項目となっている。WAI-ARIA Authoring Practices では `role="menu"` の子要素は `menuitem`（または `menuitemradio` / `menuitemcheckbox` / `separator` / `group`）のみが許容される。`<form>` 要素にはデフォルトロールがあるため、SR によってはメニュー直下に「フォーム領域」が割り込み、ログアウト menuitem の正しいインデックス（n / total）アナウンスが崩れる。テスタの aria 属性検証（H09）は属性値そのものは通過しているが、構造の妥当性までは検証していない。
- **修正案**: `<form>` 要素に `role="none"`（または `role="presentation"`）を付与し、暗黙ロールを無効化してメニュー構造から透過させる。実装例: `<form action={logout} onSubmit={close} role="none">`。1 属性追加のみで動作・スタイルへの副作用はない。
