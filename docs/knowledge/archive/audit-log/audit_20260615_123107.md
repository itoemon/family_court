## 指摘一覧

### [LOW-001] テスト用ヘルパー `loginAs` が未使用のデッドコード（tests/e2e/middleware-next.spec.ts:23-29）
- **内容**: `tests/e2e/middleware-next.spec.ts:23` に定義された `async function loginAs(page, email, password)` がファイル内のどのテストからも呼ばれていない。FEAT-MIDDLEWARE-NEXT-3 / 4 / 5（76-79 行、98-101 行、118-121 行）はそれぞれ `page.fill` / `page.click` を直接書いており、`loginAs` を経由していない。デッドコードはレビュー時のノイズとなり、後続の修正者が「使われている」と誤認する余地を残す。
- **修正案**: ヘルパーを実際に各テストで呼び出して重複を排除する（推奨）か、未使用なら削除する。

### [LOW-002] テストヘルパーの `page: any` が task.md の慣習指示に反する（tests/e2e/middleware-next.spec.ts:23）
- **内容**: task.md L57 で「既存 `tests/e2e/` の慣習に合わせる（`page: Page` 型化、hard assertion）」と明示されているが、`loginAs(page: any, ...)` は `any` を使用している。`@playwright/test` から `Page` 型を import すれば解消する。なお各 test ブロック内の `{ page }` 引数は Playwright が型を自動推論するため問題ない。本指摘は LOW-001 を削除する選択を取れば同時に解消する。
- **修正案**: `import { test, expect, type Page } from '@playwright/test';` に変更し、`loginAs(page: Page, ...)` とする。または LOW-001 と合わせてヘルパー自体を削除する。

## 総評

リード実装の差分は `middleware.ts` 内 6 行追加 / 1 行削除のみで、design.md `## FEAT-MIDDLEWARE-NEXT 対応` で挙げられた 3 つの監査観点をいずれもクリアしている。

1. **内部パス由来の保証**: `pathname + request.nextUrl.search` は `request.nextUrl`（Next.js がパース済みの内部 URL オブジェクト）由来で、外部 origin が混入する経路がない。`pathname` は常に `/` から始まる正規化済みパス、`search` は `?` 接頭辞付きクエリ文字列で、いずれもプロトコルやホスト名を含まない。
2. **無限ループの非発生**: `config.matcher`（middleware.ts:51-53）の負の先読み `(?!...|auth|...)` により `/auth/login` 自体は middleware 対象外。redirect 先がさらに redirect される構造はない。
3. **URLEncode の正しさ**: `URL` API の `searchParams.set("next", value)` が値を自動エンコードする。手動 `encodeURIComponent` を呼んでおらず、二重エンコードの懸念もない。

open redirect 観点でも二重防御が成立している。middleware 側は内部パスのみを `next` に格納し、`app/auth/login/page.tsx:36-47` の `new URL(rawNext, window.location.origin)` + `u.origin === window.location.origin` ガードが、攻撃者が `/auth/login?next=...` を直接細工してきた場合（scheme-relative URL、`javascript:` スキーム、バックスラッシュ含有等）も origin 比較で弾く設計になっており、いずれの経路でもクロスオリジン遷移は発生しない。

テスト側も CRITICAL 4 件 + FEAT-MIDDLEWARE-NEXT 5 件すべて通過済みで、要件カバレッジ（基本動作・クエリ保持・ログイン後復帰・リグレッション）に過不足はない。LOW 指摘 2 件はいずれも本番コードではなくテストファイルのヘルパーに関するもので、機能・セキュリティに影響しない軽微なクリーンアップ事項である。マージ可。
