## BUG-005 テストシナリオ

### タスク概要

**BUG-005**: 「閉廷」AI 生成アナウンスの発火タイミングを、現状の「全ラウンド完了 → `phase=extension_voting` 遷移時」から、「ユーザーが終了を確定した時 (`phase=judging` 遷移時)」へ変更

**ブランチ**: feature/20260615-163410  
**設計書**: docs/knowledge/design.md § BUG-005 閉廷アナウンス条件の修正

---

### [BUG-005-1] argument フェーズ中は closing が生成されていない

- **結果**: ✅ **通過**
- **内容**: 
  - 原告・被告が opening フェーズで発言し、argument フェーズに進む
  - この段階で `judge_messages.trigger_type='closing'` が挿入されていないことを確認
  - **検証**: Supabase admin クライアントで `judge_messages` テーブルを直接クエリ
- **期待動作**: closing は phase=judging 遷移時のみ挿入される（argument フェーズでは出ない）
- **実行時間**: 7.5秒
- **データベース検証**: ✓ 合格

---

### [BUG-005-2] 実装確認 - lib/case-closing.ts に AI 閉廷宣告ヘルパーが存在

- **結果**: ✅ **通過**
- **内容**: 
  - 実装ファイルレベルでの確認：
    - `lib/case-closing.ts` に `insertClosingJudgeMessage` ヘルパーが存在
    - このヘルパーは `judge_messages` テーブルへの INSERT のみを担当
    - 既存の `lib/greetings.ts` の `insertClosingGreetingsForCase` と分離（テーブル境界保護）
  - **grep 確認**: `trigger_type='closing'` は `lib/case-closing.ts:55` にのみ存在
- **設計準拠**: ✓ 確認済み
- **実行時間**: 0ms

---

### [BUG-005-3] 実装確認 - closing 生成は phase=judging 遷移時のみ

- **結果**: ✅ **通過**
- **内容**: 
  - 実装ファイルレベルでの確認：
    - **argument/route.ts**: closing 生成パスが削除されている
      - `nextPhase === "argument"` のみで turn judge message を生成
      - extension_voting 遷移時は judge_messages INSERT を一切しない
    - **end-proposal/route.ts**: `phase=judging` 遷移成功後に `insertClosingJudgeMessage` を呼び出し
    - **extension-vote/route.ts**: 同様に `phase=judging` 遷移後に `insertClosingJudgeMessage` を呼び出し
  - **grep 確認**: `insertClosingJudgeMessage` の呼び出しは 2 箇所（end-proposal, extension-vote）
- **設計準拠**: ✓ 確認済み
- **実行時間**: 0ms
