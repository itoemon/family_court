### CRITICAL-M01: 2ユーザー間でターン交代の会話ができる（両者認証済み）
- **結果**: ✅ 通過
- **内容**: 
  - 原告がケース作成 → 被告がアカウントでログインして参加
  - 原告が opening フェーズで最初の発言を投稿
  - 被告のターンになったことを確認してから返答
  - 原告側にも被告の返答が反映される
- **詳細**: PATCH /api/cases/:id でアカウント参加が成功、両者のターン交代が正常に機能

### CRITICAL-M02: ページリロード後もセッションが維持される
- **結果**: ✅ 通過
- **内容**:
  - ケース作成 → 被告が参加（M01 前半と同じ）
  - 原告側: リロード後も発言フォームが表示される
  - 被告側: 被告のターンになってからリロード後も発言フォームが表示される
- **詳細**: セッション Cookie が正常に保持され、認証・ロール情報が復元される

### CRITICAL-M03: 第三者認証ユーザーが被告として発言できない
- **結果**: ✅ 通過
- **内容**:
  - 原告がケース作成
  - ゲストが被告として参加（defendant_guest_name セット）
  - 認証済みユーザー B（無関係の第三者）が同じ URL を開く
  - 第三者には発言フォームが表示されない（observer 扱い）
- **詳細**: GET /api/cases/:id が callerRole="observer" を返し、フロントが フォームを非表示にする

### CRITICAL-M04: ゲスト被告が Cookie トークンで発言できる
- **結果**: ✅ 通過
- **内容**:
  - 原告がケース作成
  - ゲストコンテキスト（未ログイン）がケース URL を開く
  - 「ゲストとして参加」を選択し名前を入力
  - 参加完了 → 発言フォームが表示される（Cookie トークン付与の証拠）
  - 原告の発言 → ゲストが返答 → 原告側にも反映される
- **詳細**: PATCH /api/cases/:id { asGuest: true } で guest_defendant_* Cookie が正常に発行・機能
