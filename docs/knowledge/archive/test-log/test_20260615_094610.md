## シナリオ一覧

### [BUG-007-1] 通常ログイン（遷移先指定なし）→ / に遷移
- **結果**: ✅ 通過
- **内容**: `/auth/login` で e2e_user_a の認証情報を入力して signin → URL が / に変わることを確認
- **実行時間**: 2.8s
- **検証項目**:
  - ログイン後に /auth/login から離れる
  - 遷移先が / （またはクエリ付き）に一致する

### [BUG-007-2] ?next= パラメータ付きログイン → 指定パスに遷移
- **結果**: ✅ 通過
- **内容**: `/auth/login?next=/history` を開いて signin → /history に遷移することを確認
- **実行時間**: 295ms
- **検証項目**:
  - useSearchParams() で next クエリを正しく解釈している
  - 指定されたパスに遷移する
  - middleware による振り分けではなく、指定パスそのものに遷移する

### [BUG-007-3] パスワード誤り時にエラーメッセージ表示（リグレッション）
- **結果**: ✅ 通過
- **内容**: 誤ったパスワードを入力してログイン → エラーメッセージが表示され、URL は /auth/login のまま
- **実行時間**: 1.1s
- **検証項目**:
  - エラーメッセージ「メールアドレスまたはパスワードが違います」が表示される
  - ページリダイレクトが発生しない（/auth/login に留まる）
  - 既存のエラーハンドリングが壊れていない

### [BUG-007-4] open redirect 防御（不正な next 値はフォールバック）
- **結果**: ✅ 通過
- **内容**: `?next=//example.com/evil` など不正なパスを指定して signin → / にフォールバック、外部ドメインに遷移しない
- **実行時間**: 801ms
- **検証項目**:
  - protocol-relative URL（//example.com）で外部 origin に飛ばない
  - 同一 origin（localhost:3000）に留まる
  - セキュリティリスク（open redirect）がない
