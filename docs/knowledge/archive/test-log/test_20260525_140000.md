## B-1: `defendantId` 除去の検証

### チェック項目

| ファイル | 検証内容 | 結果 |
|---|---|---|
| `lib/types.ts` | `Case` インターフェースから `defendantId: string \| null` が削除されているか | ✅ Pass |
| `lib/case-response.ts` | 返却オブジェクトから `defendantId: c.defendant_id ?? null,` が削除されているか | ✅ Pass |
| `app/api/cases/[id]/verdict/route.ts` | `caseForClaude` オブジェクトから `defendantId: c.defendant_id ?? null,` が削除されているか | ✅ Pass |
| クライアント側（`app/` 全体） | `defendantId` を参照している箇所がないか（grep 確認） | ✅ Pass |

### 詳細

**`lib/types.ts`**

`Case` インターフェースに `defendantId` フィールドが存在しないことを確認した。現在の定義:
```ts
export interface Case {
  id: string;
  topic: string;
  callerRole?: "plaintiff" | "defendant" | "observer";
  plaintiff: Player | null;
  defendant: Player | null;
  // （defendantId フィールドなし）
  ...
}
```
設計書通り削除済み。`defendant: Player | null` オブジェクト（name・joinedAt）は残存している。

**`lib/case-response.ts`**

返却オブジェクト（65〜93行）に `defendantId` の記述がないことを確認した。`defendant` オブジェクト（`{ name, joinedAt }`）は 43行 で構築されており、UUID のみが除去されている。

**`app/api/cases/[id]/verdict/route.ts`**

`caseForClaude: Case` オブジェクト（43〜65行）に `defendantId` の記述がないことを確認した。`Case` 型に合致した形でコンパイルエラーなく実装されている。

**クライアント側 grep 確認**

`app/` ディレクトリ全体を `defendantId` で検索した結果、一致なし（No matches found）。クライアントコードが `defendantId` を参照していないことを確認した。

## B-2: ログアウトエラー通知の検証

### チェック項目

| ファイル | 検証内容 | 結果 |
|---|---|---|
| `app/actions/auth.ts` | `cookies` を import し、エラー時に `flash_error` Cookie をセットしているか | ✅ Pass |
| `app/layout.tsx` | `flash_error` Cookie を読み取り `<ErrorBanner>` を条件付きで差し込んでいるか | ✅ Pass |
| `app/components/ErrorBanner.tsx` | 新規作成済みか（Client Component・useEffect で `/api/clear-flash` fetch・× ボタン） | ✅ Pass |
| `app/api/clear-flash/route.ts` | 新規作成済みか（`maxAge: 0` で Cookie 削除） | ✅ Pass |

### 詳細

**`app/actions/auth.ts`**

`import { cookies } from 'next/headers'` が追加済み。`signOut()` エラー時に以下が実行される:
```ts
cookieStore.set('flash_error', 'logout_failed', {
  path: '/',
  httpOnly: true,
  maxAge: 30,
})
```
設計書通り `httpOnly: true`・`maxAge: 30` で実装済み。

**`app/layout.tsx`**

`import { cookies } from 'next/headers'` および `import ErrorBanner from '@/app/components/ErrorBanner'` が追加済み。`RootLayout` が `async` になっており、Cookie を読み取って `{flashError && <ErrorBanner errorCode={flashError} />}` で条件付き差し込みが実装済み。Server Component のまま維持されている。

**`app/components/ErrorBanner.tsx`**

ファイルが新規作成されており、以下を確認した:
- `'use client'` ディレクティブあり（Client Component）
- `useState` / `useEffect` を使用
- `useEffect` 内で `fetch('/api/clear-flash').catch(() => {})` を実行
- `ERROR_MESSAGES` に `logout_failed` → `'ログアウト処理でエラーが発生しました。再度お試しください。'` のマッピングあり
- × ボタン（`aria-label="閉じる"`）クリックで `setVisible(false)` 実行
- UI: `bg-rose-50 border-b border-rose-100` / テキスト: `text-rose-700` （設計書通り）

**`app/api/clear-flash/route.ts`**

ファイルが新規作成されており、GET ハンドラが `res.cookies.set('flash_error', '', { path: '/', maxAge: 0 })` で Cookie を削除する実装を確認した。
