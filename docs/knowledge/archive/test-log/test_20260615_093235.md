### 3. next パラメータの open redirect 脆弱性の有無
**検証結果: 現時点では問題なし ✅**

- Next.js の `router.push()` は内部パス前提の API のため、外部 URL が渡されても無視されるまたは内部パスとして解釈される挙動が一般的
- 本 PR では追加のサニタイズは実装していないが、task.md 監査観点の「脆弱性ありと判断した場合」の推奨（`next.startsWith("/") && !next.startsWith("//")` のガード）は、将来的な改修で検討可能
