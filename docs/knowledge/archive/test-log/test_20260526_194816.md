## 問題分析

### 根本原因：ログイン認証フロー障害

**現象**:
```
TimeoutError: page.waitForURL('/', { timeout: 10_000 })
waiting for navigation to "/" until "load"
```

**推定原因**:
1. ログインフォーム送信後、ブラウザが POST リクエストを `signInWithPassword()` に送信
2. Supabase 認証エンドポイントが応答を返す
3. ブラウザが `/` へリダイレクト
4. **しかし、ページが完全にロードされない、または middleware/useEffect で再度 `/auth/login` へリダイレクトされる可能性**

**app/page.tsx の挙動**:
- Client Component（"use client"）
- useEffect で `supabase.auth.getUser()` を呼び出し
- ユーザーが未認証の場合 `router.push("/auth/login")` で再度ログインページへリダイレクト
- ログイン直後のセッションが正しく確立されていない可能性

**考えられるシナリオ**:
```
ブラウザ: ログインボタンクリック
    ↓
Supabase Auth: signInWithPassword()
    ↓
ブラウザ: "/" へリダイレクト
    ↓
app/page.tsx (Client Component): useEffect 実行
    ↓
supabase.auth.getUser(): セッション確認
    ↓
❌ ユーザー取得失敗 → /auth/login へリダイレクト
    ↓
（リダイレクトループまたは無限待機）
```
