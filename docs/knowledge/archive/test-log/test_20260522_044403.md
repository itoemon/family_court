## CRITICAL シナリオ詳細

### [CRITICAL-M01] 2ユーザー間でターン交代の会話ができる（両者認証済み）

- **結果**: ✅ **通過**
- **実行時間**: 8.4秒
- **確認内容**:
  - ユーザー A（原告）がケースを作成
  - ユーザー B（被告）がアカウントでログインして参加
  - opening フェーズで原告→被告の順に発言が交互に送受信される
  - 各ユーザーのブラウザに相手の発言がリアルタイムに反映される（ポーリング反映）
  - ターン制御が正常に機能し、相手のターン中は textarea が非表示になる

### [CRITICAL-M02] ページリロード後もセッションが維持される

- **結果**: ✅ **通過**
- **実行時間**: 3.3秒
- **確認内容**:
  - 原告が複数回ページリロード後も認証状態とロール情報が維持される
  - 被告がページリロード後も認証状態とロール情報（defendant）が維持される
  - リロード後も発言フォーム（textarea）が適切に表示される
  - セッション Cookie が正常に保存・復元されている

### [CRITICAL-M03] 第三者認証ユーザーが被告として発言できない

- **結果**: ✅ **通過**
- **実行時間**: 2.7秒
- **確認内容**:
  - ゲストユーザーがケースに被告として参加（Cookie トークン発行）
  - 認証済み第三者（ユーザー B）が同じ URL を開いても observer 扱いになる
  - 第三者には発言フォーム（textarea）が表示されない
  - GET /api/cases/[id] の callerRole が正しく "observer" を返す（実装ノート検証）

### [CRITICAL-M04] ゲスト被告が Cookie トークンで発言できる

- **結果**: ✅ **通過**
- **実行時間**: 4.4秒
- **確認内容**:
  - ゲストユーザーが未ログイン状態でケースに参加（名前のみ入力）
  - PATCH /api/cases/[id] { asGuest: true } により guest_defendant_{id} Cookie が発行される
  - ゲストユーザーがコンテキスト維持したまま発言フォームを使用できる
  - 発言がサーバーに保存される
  - 原告がページリロード後もゲストの発言が反映される
