# 要件定義書: 家庭裁判所

## アプリ概要

恋人・夫婦・家族が、裁判形式で話し合いを行い、AI が判決を下す Web アプリ。
対立や緊張感を煽らず、「ちゃんと話し合える場」を提供することを目的とする。

---

## ユーザー種別

| 種別 | 説明 | アカウント |
| --- | --- | --- |
| 原告（Plaintiff） | 話し合いを始める側。ケースを作成する | 必須 |
| 被告（Defendant） | 招待される側 | アカウント or ゲスト（名前のみ）を選択可能 |

---

## 機能要件

### 1. 認証・アカウント管理

- メールアドレス＋パスワードでのサインアップ・ログイン
- サインアップ時に表示名（display_name）を登録する
- ログイン状態はセッションで管理する（Supabase Auth）
- ログアウト機能

### 2. API キー管理（BYOK）

- ログイン済みユーザーはプロフィール画面で Claude API キーを登録できる
- 登録されたキーは AES-256-GCM で暗号化して DB に保存する（平文保存禁止）
- キーの登録状況（登録済み / 未登録）を画面上に表示する
- キーは上書き登録できる（削除は不要）
- キーの検証: 登録時に Claude API へのテストリクエストを送り、有効かどうかを確認する

### 3. ケース作成

- ログイン済みユーザー（原告）がケースを作成できる
- 作成時に「議題」（topic）を入力する（必須・最大200文字）
- 作成されたケースには一意の URL が生成される（`/case/[id]`）
- 原告はその URL を被告に共有する

### 4. ケース参加（被告）

- 被告は共有 URL を開くとケースに参加できる
- 参加方法は2択:
  - **アカウントでログイン**: 既存アカウントで参加。履歴が残る
  - **ゲストとして参加**: 名前を入力するだけで参加できる

### 5. 話し合い（議論フロー）

話し合いは以下のフェーズで進行する:

```text
waiting → opening → argument（N ラウンド）→ closing → judging → verdict
```

| フェーズ | 内容 |
| --- | --- |
| waiting | 被告の参加を待っている状態 |
| opening | 双方が冒頭陳述を行う（原告→被告の順） |
| argument | 本論。原告→被告を1ラウンドとし、最大3ラウンド |
| closing | 双方が最終弁論を行う（原告→被告の順） |
| judging | AI が判決を生成中 |
| verdict | 判決が表示された最終状態 |

- 各発言には文字数制限を設ける（最大500文字）
- ターンは自動で交互に切り替わる（相手のターン中は入力不可）
- フェーズ・ターンの状態はリアルタイムまたはポーリングで同期する

### 6. AI 判決

- closing フェーズ終了後、原告の API キーを使って Claude API を呼び出す
- 全発言を読んで判決を生成する
- 判決の内容:
  - 勝者（原告 / 被告 / 引き分け）
  - 原告スコア（0〜100）
  - 被告スコア（0〜100）
  - 判決理由（文章）
  - 総評（文章）
- 判決は DB に保存し、verdict ページで表示する

### 7. 共通 UI

- 全ページに共通ヘッダー・フッターを表示する（実装済み）
- ヘッダー: ログイン状態に応じてナビゲーションを切り替える
- フッター: コピーライト表記

## 非機能要件

### デザイン原則

- 温かみのある・柔らかい雰囲気。対立感・緊張感を煽らない
- 配色は stone 系をベーストーンとし、赤・強調原色を避ける
- モバイルファースト・レスポンシブ対応

### セキュリティ

- Row Level Security（RLS）を全テーブルで有効化する
- API キーは必ず暗号化して保存する（平文禁止）
- サーバーサイドでのみ API キーを復号・使用する
- セッション取得は `auth.getUser()` を使用する（`getSession()` 禁止）

### 技術スタック

- Next.js 14 App Router + TypeScript + Tailwind CSS
- Supabase（PostgreSQL + RLS + Auth）
- Vercel（デプロイ）
- Claude API（Anthropic）

## 画面一覧

| 画面 | パス | 認証 |
| --- | --- | --- |
| ホーム | `/` | 必須 |
| ログイン | `/auth/login` | 不要 |
| サインアップ | `/auth/signup` | 不要 |
| プロフィール（API キー登録） | `/profile` | 必須 |
| ケース作成 | `/case/new` | 必須 |
| ケースルーム（話し合い） | `/case/[id]` | 任意（被告はゲスト可） |
| 判決 | `/case/[id]/verdict` | 不要 |

---

## DB スキーマ（概要）

```text
profiles      ← auth.users と 1対1（display_name, api_key_encrypted）
cases         ← 話し合いのケース（topic, phase, currentTurn, round, maxRounds）
              ← plaintiff_id（必須）+ defendant_id OR defendant_guest_name（排他）
arguments     ← 各ターンの発言（case_id, role, phase, round, content）
verdicts      ← AI 判決（case_id, winner, summary, reasoning, scores）
```

---

## 対象外（今回スコープ外）

- Google / GitHub OAuth ログイン
- リアルタイム通信（現状はポーリング）
- 過去ケースの一覧・履歴表示
- ゲスト参加後のアカウント紐付け
- Claude 以外の AI プロバイダー対応
