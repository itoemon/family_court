## 事前確認事項（テスト実施時の課題）

### ⚠️ テストアカウント間のフレンド関係が未設定

テストアカウント（e2e_user_a@example.com, e2e_user_b@example.com）間に `friend_requests` レコードが存在しない。

**原因**: FEAT-002 Phase 2（フレンド機能）のテストアカウント初期化スクリプトが存在しないか、実行されていない。

**対策**: 
1. Supabase ダッシュボードで以下のレコードを手動作成：
   - `friend_requests` テーブルに、`sender_id = e2e_user_a`, `receiver_id = e2e_user_b`, `status = 'accepted'` のレコード
   - 相互方向も同様に作成推奨（`sender_id = e2e_user_b`, `receiver_id = e2e_user_a` のレコード）

2. または、テストの beforeEach で自動的にフレンド関係を確立するセットアップロジックを追加
