### [NORMAL-002] hasApiKey 状態の正確性

**要件**: PATCH `/api/profile` レスポンスに `hasApiKey: boolean` が含まれ、サーバー側の現在値に基づく。

**確認内容**:
- `app/api/profile/route.ts` (PATCH) の 38 行: `hasApiKey: !!updatedProfile?.api_key_encrypted`
- 更新処理後、`select("api_key_encrypted").single()` でサーバー側の現在値を取得
- API キーが入力されていない場合でも、既存キーがあれば `hasApiKey === true`

**実装確認ポイント**:
- リクエストに `apiKey` フィールドが含まれている場合のみ暗号化して保存
- リクエストに含まれていない場合は `updates` に追加しない（既存値を保持）
- レスポンスは常にサーバー側の現在値を返す（クライアント状態に依存しない）

**結果**: ✅ **通過**

### [NORMAL-003] catch エラーメッセージの表示反映

**要件**: プロフィール保存エラー時に `err.message` が画面に表示される。

**確認内容**:
- `app/profile/page.tsx` の 57-59 行:
  ```typescript
  catch (err: unknown) {
    const message = err instanceof Error ? err.message : "保存中にエラーが発生しました";
    setMessage(message);
  }
  ```
- API レスポンスが `{ error: "..." }` の場合、`throw new Error(data.error)` で例外化
- catch ブロックで `err.message` を `setMessage` に反映
- メッセージは画面上に 115-119 行の条件付きレンダリングで表示

**実装確認ポイント**:
- 無効 API キー時: 400 エラー + `{ error: "APIキーが無効です..." }`
- その他エラー時: 500 エラー + `{ error: "..." }`
- クライアント側で `data.error` を `Error` にラップして catch に送出

**結果**: ✅ **通過**
