## 考察：Header 刷新との因果関係

### 観察事項

1. **ログイン画面専用テスト（`/auth/login`）が軒並み失敗している**
   - handoff では「Header.tsx のリファクタ」と記載され、認証フローの変更は明記されていない
   - しかし、事実として `input[type="email"]` が見つからない

2. **Header コンポーネント刷新による影響の可能性**
   - `app/components/Header.tsx`：Server Component のリファクタ（`HeaderUserMenu.tsx` に機能を移譲）
   - `app/components/HeaderUserMenu.tsx`：Client Component の新規作成（ドロップダウン管理）
   - layout.tsx での `<Header />` 呼び出し位置は不変とされているが、レンダリング遅延の可能性

3. **app/layout.tsx のマウント順序**
   - Header が Client Component（`HeaderUserMenu`）を使用することで、ハイドレーションが発生
   - ログインページ自体が `/auth/login` で Header を含むレイアウトを使用している場合、初期表示遅延が発生
   - `page.waitForSelector('input[type="email"]')` のタイムアウト時間（10,000ms）を超える可能性
