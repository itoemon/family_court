## FEAT-004 シナリオ（法律Hub・公開・インポート機能）

### [FEAT-004-E01] 公開→Hub出現→インポート→元法律不変

- **結果**: ✅ **通過**
- **実行時間**: 4.288 秒
- **内容**:
  - ユーザーA が法律を新規作成 (`is_public=false` で作成)
  - A が公開トグルを ON → `is_public=true` に更新
  - ユーザーB が `/laws/hub` にアクセス → A の法律が一覧に出現
  - B がインポートボタンをクリック
  - 新規法律が B のオーナーとして作成される
  - インポート元の A の法律は不変（owner_id・name・article 変化なし）
  - `laws_select_public` ポリシーが機能している

---

### [FEAT-004-E02] 非公開は非出現・非公開化で消える

  - 非公開法律（`is_public=false`）は Hub 一覧に出ない
  - 公開状態の法律を非公開に変更
  - Hub 一覧から消える（RLS が即座に反映）
  - 公開・非公開の状態変更が正常に動作

## 検証観点

### ✅ セキュリティ

- **RLS**: `laws_select_public` ポリシー追加により公開法律のみ非メンバーに可視
- **認可**: visibility / import は所有者・公開状態で確実にガード
- **情報漏洩なし**: Hub API は `owner_id` を返さず（`owner_display_name` のみ）

### ✅ データ整合性

- **純クローン**: インポート時に元法律が変更されない（owner_id・name・article 不変）
- **新規法律初期化**: インポート後、新規法律のオーナー=インポーター、メンバー登録完了
- **migration 冪等性**: `ADD COLUMN IF NOT EXISTS` + `DROP POLICY IF EXISTS → CREATE` で重複実行安全

### ✅ UI/UX

- **公開トグル UI**: `/laws/[id]` ページのオーナー向けセクションに「Hub に公開する」「非公開にする」が配置
- **Hub ページ**: 公開法律一覧・検索・条文プレビュー・インポートボタンが正常に機能
- **導線**: `/laws` → `/laws/hub` への遷移が可能

### ✅ リグレッション

- **CRITICAL-M01〜M04**: ケース・会話機能に退行なし
- **既存 `laws.spec.ts`**: `laws_select_member_or_invitee` ポリシー未変更により非メンバー保護維持
