## 指摘一覧

### [HIGH-001] PR #44 HEAD (3fbc528) に open redirect 防御が含まれておらず、BUG-007-4 テストの根拠と矛盾する（app/auth/login/page.tsx:31-32 in HEAD）
- **内容**:
  `git diff main...HEAD -- app/auth/login/page.tsx` で確認できる commit `3fbc528` の差分には `next` パラメータのサニタイズが一切実装されていない。コミット済みの該当箇所は以下の 2 行のみで、`searchParams.get("next")` の戻り値をそのまま `router.push()` に渡している。

  ```tsx
  // commit 3fbc528 の HEAD（マージされる差分）
  const next = searchParams.get("next") || "/";
  router.push(next);
  ```

  この実装は `?next=//evil.example.com/path` のような protocol-relative URL を内部パスとして解釈する。Next.js App Router の `router.push` は origin が現在の origin と異なる場合に `window.location` フォールバック経路で外部 navigation するため、認証直後に外部ドメインへ飛ばされる open redirect 経路が成立する（CWE-601）。認証成功直後のリダイレクトに乗せたフィッシング攻撃（攻撃者のサイトで再ログインさせて credential を窃取）に悪用可能。

  **重大な事実**: working tree（`git status: M app/auth/login/page.tsx`）には URL.origin ベースのガード（app/auth/login/page.tsx:36-47）が追加済みだが、これは uncommitted のままで PR #44 には含まれていない。`docs/knowledge/handoff/test-to-aud.md` の BUG-007-4 通過記録（2026-06-15 09:46:10）は、この uncommitted な working tree に対する実行結果であり、PR HEAD `3fbc528` をそのままマージすると **BUG-007-4 が即座に回帰し脆弱性が production に出る**。`docs/knowledge/task.md` の監査観点 3「`next` パラメータの open redirect 脆弱性」が明示的に指摘している論点でもある。

- **修正案**:
  working tree に既にある以下のガードを `fix/bug007-login-redirect` ブランチに `git add app/auth/login/page.tsx && git commit` して PR #44 に含めること。前方一致ガード（`next.startsWith("/") && !next.startsWith("//")`）よりもこの URL ベース実装の方が backslash・`%2f` 等のバイパス余地を塞げるため、working tree 実装をそのまま採用するのが妥当。

  ```tsx
  const rawNext = searchParams.get("next");
  let next = "/";
  if (rawNext) {
    try {
      const u = new URL(rawNext, window.location.origin);
      if (u.origin === window.location.origin) {
        next = u.pathname + u.search + u.hash;
      }
    } catch {
      // 不正な URL は "/" にフォールバック
    }
  }
  router.push(next);
  ```
