## 指摘一覧

### [LOW-001] `useSearchParams()` が Suspense 境界に包まれていない（app/auth/login/page.tsx:10）
- **内容**: `app/auth/login/page.tsx` は `"use client"` で `LoginPage` 全体が Client Component だが、`useSearchParams()` を直接呼び出している（10 行目）。Next.js 16 の公式ガイダンスでは `useSearchParams()` を使う Client Component は最寄りの祖先で `<Suspense>` でラップすることが推奨されており、未ラップだと静的生成時に CSR バウンドアウト（route 全体がクライアント側レンダリングに退避）する。`app/layout.tsx:44` の `<Suspense>` は `<Header />` のみを包んでおり、`{children}` 配下（=login ページ）には Suspense 境界が存在しない。

  既存の `app/case/[id]/CaseRoom.tsx` も同じパターンで動いており、テスタ実行で build エラー・ランタイム警告が観測されていないこと、`/auth/login` は `"use client"` 全体で初めからクライアント側レンダリングであり静的化されていないことから、現時点で実害は発生していない。task.md の監査観点 #2 で「念のため確認」とされている項目の事実確認として LOW で記録する。

- **修正案**: 任意対応。実害が出るまで放置でも問題ないが、推奨対応としては `app/auth/login/page.tsx` の `LoginPage` 中身を別の Client Component（例: `LoginForm`）に分離して、`page.tsx` 側で `<Suspense fallback={...}><LoginForm /></Suspense>` の形でラップする。これにより将来 Next.js の静的最適化が強化された際に build 警告が出ることを予防できる。今 PR のスコープには入っていないため、別タスクとして backlog に積む扱いでよい。
