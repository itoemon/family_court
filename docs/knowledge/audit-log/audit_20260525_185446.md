## 指摘一覧

### [LOW-001] `createSessionClient()` が try-catch の外にある（`defense/draft/route.ts:26`）

- **内容**: `defense/draft/route.ts` の 26 行目（`const session = await createSessionClient()`）は try-catch で保護されていない。他の全 Route は `createSessionClient()` を try-catch で包み、例外時に制御された 500 レスポンスを返している（`argument/route.ts:26–48`、`defense/route.ts:15–29`、`route.ts:25–48`）。本 PR で同ファイルへ `verifyGuestToken` の try-catch（39–49 行）が追加されたが、その上流の `createSessionClient()` が対象外となっている。`createSessionClient()` が例外をスローした場合、Next.js のデフォルトエラーハンドラが動作し、開発環境ではスタックトレースが露出しうる。本番（Vercel）では非表示だが、挙動が他 Route と一貫しない。
- **修正案**: 既存 Route のパターンに倣い、26–27 行目を try-catch で包む。

```typescript
let user;
try {
  const session = await createSessionClient();
  const { data: { user: u } } = await session.auth.getUser();
  user = u;
} catch (err) {
  console.error("createSessionClient failed:", err);
  return NextResponse.json(
    { error: "サーバー設定エラーが発生しました。管理者に連絡してください。" },
    { status: 500 }
  );
}
```

### [LOW-002] `guest_tokens.token_hash` に UNIQUE 制約なし（`supabase/migrations/20260525000003_add_guest_tokens.sql`）

- **内容**: `guest_tokens` テーブルに `token_hash` の一意性制約が存在しない。HMAC-SHA256 の衝突確率は実用上ゼロだが、UNIQUE 制約がなければ、アプリバグ（例: nonce 生成が壊れてゼロ値を返す）によって同一ハッシュが複数 INSERT された際に DB レベルで検知できない。また、`token_hash` の重複が許容されると、`verifyGuestToken` の SELECT が複数件ヒットする状態（設計では想定しない）が発生しうる（`.limit(1)` で吸収されてはいる）。
- **修正案**: マイグレーションに UNIQUE 制約を追加する。`case_id` インデックスは残し、`token_hash` 単体に UNIQUE を設けることを推奨する（異なるケースで同一 nonce が生成される確率はゼロだが、DB の一貫性保証として有効）。

```sql
CREATE UNIQUE INDEX ON guest_tokens(token_hash);
```
