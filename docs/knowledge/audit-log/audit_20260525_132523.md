### B-2: ログアウトエラー通知

#### `app/actions/auth.ts` — `flash_error` Cookie が `httpOnly: true` でセットされているか

**結果**: ✅ 確認済み

```ts
cookieStore.set('flash_error', 'logout_failed', {
  path: '/',
  httpOnly: true,
  maxAge: 30,
})
```

`httpOnly: true` が明示指定されており、XSS スクリプトによる Cookie 読み取りを防ぐ。`maxAge: 30`（秒）で短命な Cookie となっており、セッションを跨いで持続しない設計が意図どおり実装されている。`secure` フラグは Next.js が本番環境で自動付与するため、明示なしでも問題ない。

#### `app/api/clear-flash/route.ts` — GET ハンドラのみ定義されているか

**結果**: ✅ 確認済み

```ts
export async function GET() {
  const res = NextResponse.json({ ok: true })
  res.cookies.set('flash_error', '', { path: '/', maxAge: 0 })
  return res
}
```

`GET` ハンドラのみが定義されており、`POST`・`DELETE` 等の不要メソッドは存在しない。Next.js は未定義メソッドに対して 405 Method Not Allowed を自動返却する。

`/api/clear-flash` はレート制限なしに呼び出せるが、削除されるのは呼び出し元の `flash_error` Cookie のみ（Cookie は per-user）であり、他ユーザーへの影響はない。悪用シナリオのリスクは軽微と判断する。

#### `ErrorBanner.tsx` — XSS リスクがないか

**結果**: ✅ 確認済み

`errorCode` を `ERROR_MESSAGES[errorCode] ?? 'エラーが発生しました。'` でマッピングし、JSX の `{message}` としてテキストノードに描画している。React の JSX は文字列を自動エスケープするため、XSS リスクは存在しない。未知のコードに対するフォールバックメッセージ（`'エラーが発生しました。'`）も設定済み。

`app/layout.tsx` に `'use client'` ディレクティブがないことを確認した。`RootLayout` は Server Component のまま維持されており、`ErrorBanner` のみが Client Component として独立している。

#### フラッシュ Cookie の `maxAge: 30` が短命であることの確認

**結果**: ✅ 確認済み

`app/actions/auth.ts` にて `maxAge: 30` が設定されており、30秒で自動失効する。`ErrorBanner` の `useEffect` が `/api/clear-flash` を fetch することで即座に削除される二重の仕組みとなっており、意図した動作である。

## 指摘一覧

### [LOW-001] `/api/clear-flash` の Cookie 削除に `httpOnly: true` が指定されていない（`app/api/clear-flash/route.ts:5`）

- **内容**: `auth.ts` でセットする際に `httpOnly: true` を指定しているのに対し、`clear-flash/route.ts` での削除時には `httpOnly` オプションが指定されていない。`{ path: '/', maxAge: 0 }` のみである。
  
  Cookie を上書き削除する `Set-Cookie` ヘッダーは、元の Cookie のフラグと一致しないブラウザ実装によっては削除が正常に機能しない場合がある（RFC 6265 では path と domain のみが一致条件だが、実装依存の挙動が存在する）。現在の主要ブラウザ（Chrome・Firefox・Safari）では `httpOnly` の不一致で削除が妨げられることはない。したがって即時影響はないが、一貫性の観点から指摘する。

- **修正案**:
  ```ts
  res.cookies.set('flash_error', '', { path: '/', maxAge: 0, httpOnly: true })
  ```
  セット時と削除時でオプションを統一する。
